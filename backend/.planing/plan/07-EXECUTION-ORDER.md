### M1 — 基础骨架 + AI 网关就绪

| 验收项 | 验证方法 | 通过标准 |
|--------|---------|---------|
| 项目编译 | `npm run build` | 零错误 |
| 代码质量 | `npm run lint` | 零警告 |
| 测试覆盖 | `npm run test` | 覆盖率 > 80% |
| 进程存活 | `curl /health/alive` | 200 OK |
| DB 就绪 | `curl /health/ready` | `{ db: "connected" }` |
| 用户注册 | `POST /auth/register` | 返回 JWT pair |
| 用户登录 | `POST /auth/login` | 返回 JWT pair + Set-Cookie |
| Token 刷新 | `POST /auth/refresh` | 新 JWT pair |
| AI 增强 | `POST /ai/enhance` | 返回增强结果 |
| Provider 列表 | `GET /providers/active` | 返回活跃 Provider |
| 分类数据 | `GET /meta/categories` | 返回种子分类 |
| 错误格式 | 发送非法请求 | `{ success: false, error: { code, message } }` |
| 限流 | 超限请求 | 429 + `Retry-After` 头 |
| 黑名单 | 封禁 IP 后请求 | 403 + `BLACKLIST_IP_BLOCKED` |

**验收人**：开发者自验 + 手工 curl 测试
**回滚点**：Phase 1 全部代码在独立 `backend/` 目录，不影响现有系统

### 并行化机会（可缩短 ~2 周）

1. **Phase 1 内部**：错误系统（P1.07-09）‖ 日志系统（P1.10-11）‖ 指纹中间件（P1.22）
2. **Phase 1 内部**：AI 适配器（P1.32）‖ Provider Service（P1.29）‖ AI 日志（P1.28）
3. **Phase 2 内部**：分析模块（P2.01-08）‖ 部署（P2.15-18）
4. **Phase 2 内部**：4 端客户端迁移（P2.10-13）理论可并行（但建议串行以控制风险）
5. **Phase 3 可与 Phase 2 部分并行**：Sprint 3.1 不依赖 Phase 2 的客户端迁移
6. **Phase 4 内部**：Trending（P4.01-02）‖ Admin（P4.04-05）‖ SSO 页面（P4.09-10）
7. **Phase 5 内部**：多个 Hook 迁移可并行（P5.04-P5.08）
